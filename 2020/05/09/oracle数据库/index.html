结构化查询语言–SQL概述

SQL是在关系数据库上执行数据操作、检索及维护所使用的标准语言，可以用来查询数据，操作数据，定义数据，控制数据

所有数据库都使用相同或者相似的语言

数据定义语言(DDL)

用于建立、修改、删除数据库对象

  • CREATE:创建表或其他对象的结构
  • ALTER:修改表或其他对象的结构
  • DROP:删除表或其他对象的结构
  • TRUNCATE:删除表数据、保留表结构

数据操作语言(DML)

用于改变数据表中的数据;和事务相关，执行完后需要经过事务控制语句提交后才真正的将改变应用到数据库中

  • INSERT:将数据插入到数据表中
  • UPDATE:更新数据表中已存在的数据
  • DELETE:删除数据表中的数据

事务控制语言(TCL)

用来维护数据一致性的语句

  • COMMIT:提交，确定已经进行的数据改变
  • ROLLBACK:回滚，取消已经进行的数据改变
  • SAVEPOINT:保存点，使当前的事务可以回退到指定的保存点，便于取消部分改变

数据查询语言(DQL)

用来查询所需要的数据

  • SELECT语句

数据控制语言(DCL)

用于执行权限的授予和收回操作

  • GRANT:授予，用于给用户或角色授予权限
  • REVOKE:用于收回用于或角色已有的权限

oracle数据结构

  1. NUMBER(P[,S]):数字类型
    P:总位数,范围:1-38
    S:小数位数

    没有设置S,则默认值为0
    P可以设置为 *，即为38

  1. CHAR(N [BYTE/CHAR]):固定长度的字符类型
    N:占用字节数,最大长度为2000字节,默认为字节，可指定字符(一个字母占一个字节，中文字符按编码不同占2-4个字节)

    存放定长字符，存不满补空格;浪费空间，节省时间;可以不指定长度，默认为1

  1. VARCHAR2(N):变长的字符串类型
    N:最多可占用字节数,最大长度为4000字节

    变长:根据保存的数据长度，占用的空间是变化的;必须指定长度

  1. LONG和CLOB类型
    a. LONG:VARCHAR2加长版，存储变长字符串，最多达2GB的字符串数据
    b. LONG的限制:每个表只能有一个LONG类型列;不能作为主键;不能建立索引;不能出现在查询条件中…
    c. CLOB:存储定长或变长字符串，最多达4GB的字符串数据
    d.oracle建议开发中使用CLOB替代LONG类型
  1. DATE:定义日期时间的数据
    默认格式:DD-MON-RR

    DATE类型在数据库中的存储固定为7个字节，格式为:世纪+100 年 月 日 小时+1 分+1 秒+1

  1. TIMESTAMP:常用的日期类型
    a. 与DATA的区别是不仅可以保存日期和时间，还能保存小数秒，最高精度可以到ns(纳秒)
    b. 数据库内部用7或11个字节存储;精度为0,用7字节存储，精度大于0，则用11字节存储
    格式为:

    1-7:与DATE相同
    8-11:纳秒，内部运算类型为整形
    create table test(c1 DATE,c2 TIMESTAMP);

  • NOT NULL

修改表

修改表名

  • RENAME 原表名 TO 新表名;

增加列

  • ALTER TABLE 表名 ADD(字段);
    列只能增加在最后，不能插入到现有的列中

删除列

  • ALTER TABLE 表名 DROP(字段名);
    删除字段需要从每行中删掉该字段占据的长度和数据，并释放在数据块中占据的空间，如果表记录比较大，删除字段可能需要比较长的时间

DML语句

执行DML操作后，需要再执行commit语句才算真正确认了此操作

1)INSERT语句
    INSERT INTO 表名[(字段[,字段,字段,...])] VALUES(value,value,...);
        插入行
            #如果插入的列有日期字段，需要考虑日期的格式，默认格式 'DD-MON-RR'
            #可使用TO_DATE函数自定义日期格式
                例:
                    INSERT INTO 表名(...) VALUES(..,..,TO_DATE('2020-02-12','YYYY-MM-DD'));

2)UPDATE语句
    UPDATE 表名 SET 属性=value [WHERE 属性=value];
        更新表中的记录。WHERE子句用来筛选，没有则修改全表数据

3)DELETE语句
    DELETE FROM 表名 [WHERE 条件];
        删除表数据
            #在DDL语句中的TRUNCATE语句同样有删除表数据的作用
                TRUNCATE语句和DELETE语句区别:
                    a. DELETE可以有条件删除，TRUNCATE将表数据全部删除
                    b. DELETE是DML语句，可以回退;TRUNCATE是DDL语句，立即生效，无法回退
                    c. 如果是删除全部表记录，且数据量较大，DELETE效率比TRUNCATE语句低

字符串函数

CONCAT和 ||
    a. CONCAT(c1,c2)
        返回两个字符串连接后的结果
    b. 等价操作: 连接操作符 ||
    c.如果c1和c2任何一个为NULL,相当于连接了一个空格
        例:
            SELECT CONCAT(CONCAT(ename,':'),sal) FROM emp;

            SELECT ename || ':' sal FROM emp;


LENGTH
    a. LENGTH(字段)
        用于返回字符串的长度
            #类型为VARCHAR2,返回实际长度；为CHAR,则返回最大长度
        例:
            SELECT job,LENGTH(job) FROM emp;


UPPER、LOWER和INITCAP
    a.大小写转换函数:
        UPDATE(字段)  将字符串转换为大写
        LOWER(字段)      将字符串转换为小写
        INITCAP(字段) 将字符串每个单词首字母大写，其它小写
    b.如果输入的参数为空，仍然返回空
        例：
            select upper('hello world'),lower('hello world'),INITCAP('hello world') FROM dual;
        #dual是内部表 具体见百度


TRIM、LTRIM、RTRIM
    a. 截去字符串
        TRIM([c2 FROM ]c1)  从c1两端去除c2
        LTRIM(c1[,c2])        从c1左边去除c2
        RTRIM(c1[,c2])        从c1右边去除c2
            #没有可选参数即为去除空格


LPAD、RPAD
    a.补位函数，如c1不足n位，用c2重复补齐到n位;如c1超过n位，将c1从后往前截取直到n位
        LPAD(c1,n,c2) 将c2从c1左侧补齐
        RPAD(c1,n,c2) 将c2从c1右侧补齐


SUBSTR
    a.获取子串，返回c中从m位开始取n个字符(下标从1开始)
        SUBSTR(c,m[,n])
            #m=0 则从首字符开始。m为负数，则从尾部开始(不是反转)
            #没有参数n，或者n的长度超过c的长度，则取到字符串结尾为止

INSTR
    a.返回c2在c1中的位置(下标从1开始)
        INSTR(c1,c2[,n[,m]])
            #n:从n位开始搜索，默认唯一
            #m:指定子串第m次出现的位置，默认为1
            #没有找到子串c2，返回0

数值函数

ROUND
    a. 四舍五入
        ROUND(n[,m])
            #n为被处理的数字，m控制保留几位
            #m取0或没有m，则四舍五入到整数位
            #m取正数，则四舍五入到小数点后m位
            #m取负数，则四舍五入到小数点前m位


TRUNC
    a.截取数字
        TRUNC(n,[,m])
            #n,m的定义和ROUND相同，不过是按截取的方式处理数字


MDO
    a. 返回m除以n后的余数
        MOD(m,n)
            #n为0，则直接返回m


CEIL和FLOOR
    a. 截取小数部分
        CEIL(n)        截去小数，整数进一位
        FLOOR(n)    截取小数，整数不变

日期操作

日期关键字
    1) SYSDATE
        a.本质是一个oracle的内部函数，返回当前的系统时间，精确到秒
        b.默认显示格式: DD-MON-RR

    2) SYSTIMESTAMP
        a.内部函数，返回当前系统日期和时间，精确到毫秒


日期转换函数
    TO_DATE
        a. 将字符串按照指定格式转换为日期类型
            TO_DATE(c[,fmt[,nlsparams]])
                #日期格式表:
                    YY        两位数年份
                    YYYY    四位数年份
                    MM        两位数月份
                    MON        简拼月份
                    MONTH    全拼月份
                    DD        两位数的日
                    DY        周几的缩写
                    DAY        周几的全拼
                    HH24      24小时制的小时
                    HH12    12小时制的小时
                    MI        显示分钟
                    SS        显示秒

    TO_CHAR
        a.将其他类型的数据转换为字符类型
            TO_CHAR(date[,fmt[,nlsparams]])
                #将日期类型数据date按照fmt格式输出字符串，nlsparams用于指定日期语言
                    例:
                        select to_char(sysdate,'YYYY"年"MM"月"DD"日"')


日期常用函数            
    LAST_DAY
        a.返回日期date所在月的最后一天
            LAST_DAY(date)

    ADD_MONTHS
        a.返回日期deate加上i个月后的日期值
            ADD_MONTHS(date,i)

    MONTHS_BETWEEN
        a.计算date1和date2两个日期值之间隔了多少个月
            MONTH_BETWEEN(date1,date2)    
                #实际运算是date1-date2，如果date2比date1晚，会得到负值
                #除非两个日期间隔是整数月，否则会得到带小数位置的结果

    NEXT_DAY
        a.返回date日期数据的下一个周几，由char决定
            NEXT_DAY(date,char)
                #插入参数可用数字1-7表示周日-周六

    LEAST、GREATEST
        LEAST(obj1[,obj2[,obj3]]....)     返回参数中的最小值
        GREATEST(obj1[,obj2[,obj3]]....) 返回参数中的最大值
            #参数类型必须一致
            #在比较之前，参数表中第二个及以后的参数会被隐式的转换为第一个参数的数据类型，如可以转换则继续，不能转换将会报错

    EXTRACT
        a. 从datetime中提取date指定的数据，如提取年、月、日
            EXTRACT(date from datetime)

空值操作

NULL的含义
    a.数据库里的重要概念:NULL,即空值
    b.有时表中的某些字段值，数据未知或暂时不存在，取值NULL
    c.任何数据类型均可取值NULL

插入NULL值
    例:
        create table test(id number(4),name char(5));
        insert into text values(1001,NULL)//显式插入NULL值
        insert into text(id) values(1001)//隐式插入NULL值

更新成NULL值
    update text set name=NULL;
        #更新为NULL只能在此列没有非空约束的情况下才可操作
        #如果有非空约束，则无法更新，会报错

NULL条件查询
    a. NULL不等于任何值，所以:
        select * from text where name is NULL;//不能为=号，why？ 见a

空值函数
    NVL
        a. 如果expr1为NULL，则取expr2的值，expr1不为NULl，则取expr1的值;相当于else
            NVL(expr1,expr2)
                #两个参数可以是任何数据类型，但都要为同一种数据类型

    NVL2
        a.如果expr1不为NULL,取expr2的值，否则取expr3的值
            NVL(expr1,expr2,expr3)

基础查询

基本查询语句
    1) FROM子句
        a. select colunm [alias]... from table;
            select用于指定要查询的列
            from指定从哪个表查询
        b.如果要查询所有列，select后面使用 *
        c.查询特定列，select后面指定列名，列明之间逗号隔开

    2) 使用别名
        a.可以通过使用列的别名改变标题的显示样式
        b.使用方法: 列的别名跟在列名后面，空格隔开，可以使用 as 关键字，也可以不使用
        c.如果希望别名区分大小写，或包含字符或空格，则必须用双引号引起来

    3) WHERE子句用
        a. 在select语句中使用where子句来限制查询结果

查询条件
    1)逻辑判断符
        >    <    >=    !=    <>    =
        #!=等价于<>

    2)AND、OR关键字
        AND 表示 与
        OR     表示 或

    3)LIKE条件(模糊查询)
        a. 查询时不能完全确定某些信息的查询条件，或者只知道信息的一部分，可以使用LIKE
            LIKE需要借助两个通配符:
                %    表示0到多个字符
                _    表示单个字符
                    例:
                        select ename from emp where ename like '%n%';

    4) IN和NOT IN
        a.比例操作符，用来筛选符合列表范围中的数据，用另一列的值来限定要查询的列
            IN        表示在此范围
            NOT IN    表示不再此范围
                例:
                    select enmae from emp where worktime in(12193,11908);

    5) BETWEEN...AND...
        a.BETWEEN...AND...操作符用来查询符合某个值域范围条件的数据
        b.数字、字符、日期类型都适用
            例:
                select enmae from emp where worktime between 11000 and 12000;

    6) IS NULL 和 IS NOT NULL
        a.null不等于任何值，比较的时候不能用 = ，必须使用 is

    7) ANY 和 ALL
        a.ANY 和 ALL不能单独使用，需配合 > >= < <= 一起使用
            > ANY    大于最小
            < ANY     小于最大
            > ALL    大于最大
            < ALL   小于最小
                例:
                    select ename from emp where worktime <any(12000,13000);

    8) 查询条件中使用表达式和函数
        a.当需要对选出的字段进行进一步计算，可以在数字列上使用算术表达式 + - * /

    9) 使用DISTINCT过滤重复
        a.执行查询操作时，会默认显示所有行，不管查询结果是否有重复数据，去掉重复值，使用DISTINCT实现
            例:
                select distinct depton from emp;


排序    
    1) 使用ORDER BY 子句
        a.对数据按升序或降序排序，默认为升序(asc)    asc/desc
        b.ORDER BY 子句必须出现在select语句最后
        c.可以多列排序
            例:
                select * from emp order by ename asc;

聚合函数

什么是聚合函数
    查询时需要做一些数据统计，比如:查询职员表中各部门职员的平均薪水，各部门的员工人数，需要统计的数据并不能在职员表里直观列出，而是需要根据现有的数据计算得到结果，这种功能可以使用聚合函数来实现，即:将表的全部数据划分为几组数据，每组数据统计出一个结果，因为是多行数据参与运算返回一行结果，也称分组函数、多行函数、集合函数

    1) MAX 和 MIN
        a.用来取得列或表达式的最大、最小值
        b.可以用来统计任何数据类型
            例:
                select min(worktime) from emp;

    2) AVG 和 SUM
        a.统计列或表达式的平均值和和值
        b.只能操作数字类型，忽略NULL值

    3) COUNT
        a.用来计算表中的记录条数，忽略NULL值
            例:
                select count(ename) from emp;

聚合函数忽略NULL值

分组

1) GROUP BY子句
    a.当希望得到每个部门的平均薪水，而不是整个公司的平均薪水，可以把整个数据表按部门划分成一个个小组，每个小组包含一行或多行数据，在每个小组中再使用分组函数进行计算，每组返回一个结果，划分的小组有多少，最终结果集行数就有多少

例:
    select max(id),job from emp group by job;

2) HAVING 子句
    a.HAVING 子句用来对分组后的结果进一步限制，比如按部门分组后，得到每个部门的最高薪水，可以继续限制输出结果
    b. 必须跟在GROUP BY后面，不能单独存在
        例:
            select depton,max(sal) max_sal from emp group by depton having max(sal)>4000;

查询语句执行顺序

查询语句的执行循序依下列子句次序:
    1) from 子句
        执行顺序为从后往前、从右到左
            #数据量较少的表尽量放在后面

    2) where 子句
        执行顺序为自下而上、从右到左
            #将能过滤掉最大数量记录的条件写在where子句的最右

    3) group by
        执行顺序从左往右分组
            #最好在group by前使用where将不需要的记录在group by之前过滤掉

    4) having
        having子句消耗资源，尽量避免使用，having会在检索出所有记录之后才对结果集进行过滤，需要排序等操作

    5) select 子句
        少用*,尽量取字段名称，oracle在解析的过程中，通过查询数据字典将*依次转换成所有的列名，消耗时间

    6) order by 子句
        执行顺序为从左到右排序，消耗资源
