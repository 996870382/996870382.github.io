oracle约束

发表于 2020-05-09 | 分类于 oracle数据库 |

约束的作用

约束(constraint)的全程是约束条件，也称作完整性约束条件
约束是在数据表上强制执行的一些数据校验规则，当执行DML操作时，数据必须符合这些规则，如果不符合则无法执行
约束条件可以保证表中数据的完整性，保证数据间的商业逻辑

约束类型

非空约束 NN
唯一性约束 UK
主键约束 PK
外键约束 FK
检查约束 CK

非空约束

a.not null,用于确保字段值不为空
    当执行insert，必须提供这个列的数据
    当执行update，不能给这个列的值设置为null
b.如何取消非空约束
    alter table 表名 modify(字段名 null);//修改非空约束

唯一性约束

什么是唯一性约束
    a.唯一性约束条件用于保证字段或字段的组合不出现重复值
    b.当给表的某个列定义了唯一约束，该列的值不允许重复，但允许null值
    c.唯一性约束可以在建表时建立，也可以在建表后建立

添加唯一性约束
    1) 建表时 定义字段后最后一句添加唯一约束:
        create table 表明(.......,CONSTRAINT 约束名 UNIQUE(字段名，[字段名..]))
    2)alter table 表名 add constraint 约束名 unique(字段名);

主键约束

主键约束的意义
    a.主键(Primary Key)约束条件从功能上看相当于非空且唯一的组合
    b.主键字段可以是单个字段或多个字段组合，即:在主键约束下的单字段或多字段组合上不允许有空值，也不允许有重复值
    c.主键可以用来在表中唯一的确定一行数据
    d.一个表上只允许建立一个主键，而其它约束条件则没有明确的个数限制

主键选取的原则
    1) 主键应是对系统无意义的数据
    2) 永远也不要更新主键，让主键除了唯一标识一行之外，再无其它用途
    3) 主键不应为动态变化的数据，如时间戳
    4) 主键应自动生成，不要人为干预，以免使它带有除了唯一标识一行以外的意义
    5) 主键尽量建立在单列上

添加主键约束
    1) 建表时添加主键约束，在字段后加上 PRIMARY KEY关键字
    2) alter table 表名 add constraint 约束名 primary key(字段名);

外键约束

外键约束的意义
    a.外键约束条件定义在两个表的字段或一个表的两个字段上，用于保证相关两个字段的关系

添加外键约束
    a.添加外键约束需要在建表后添加
        alter table 表名 add constraint 约束名 foreign key (外键字段名) References 主表的表名(主键字段名);

外键约束对一致性的维护
    a.外键约束条件包括两个方面的数据约束:
        1) 从表上定义的外键列值，必须从主表被参照的列值中选取，或者为NULL
        2) 当主表参照列的值被从表参照时，主表的该行记录不允许被删除                                         
        例:假如emp表有主外键约束
            insert into emp(eid,name,deptno) values(1234,'dd',50);
            //失败，不存在部门50
            delete from dept where deptno=40;
            //失败，部门40被参照，不允许删除

外键约束对性能的降低
    a.如果在一个频繁DML操作的表上建立外键，每次DML操作，都将导致数据库自动对外键所关联的对应表做检查，产生开销，如果已在程序中控制逻辑，这些判断将增加额外负担，可以省去
    b.外键确定了主从表的先后生成关系，有时会影响业务逻辑

关联不一定需要外键约束
    1) 保证数据完整性可由程序或触发器控制
    2) 简化开发，维护数据时不用考虑外键约束
    3) 大量数据DML操作时不需要考虑外键，耗费时间

检查约束

什么是检查约束
    a.检查约束条件用来强制在字段上的每个值都要满足Check中定义的条件
    b.当定义了Check约束的列新增或修改数据时，数据必须符合Check约束中定义的条件

添加检查约束
    a.检查约束需要在创建表后添加
        例：员工的薪水必须大于2000
            alter table emp add constraint 约束名 check(sal>2000);
